## 2. What is NOT in the project (so it was not documented)

To avoid inventing functionality, the following were explicitly confirmed
**absent** and therefore excluded from the README/CHANGELOG:

- **No backend / server / database.** The extension is 100% client-side; it only
  reads CDD's own API responses and (for dose-response) writes back through
  CDD's own endpoints.
- **No background service worker.** `manifest.json` declares no `background` key.
- **No bespoke authentication.** The only "auth" is reusing CDD's existing
  session cookies + CSRF token in `src/content/api/cdd-api.js`.
- **No tests, no linter, no CI, no `dev`/watch script.** `package.json` exposes
  only `build:content`, `build:inject`, `build`.
- **No telemetry / remote code.** Manifest declares `data_collection_permissions
  â†’ required: ["none"]`; all logic is bundled locally by Vite.
